# 학생 상태 조회 연계 예제 프로젝트

### 프로젝트 개요
외부 학생 관리 서버에서 제공하는 학생 출석 상태 데이터를 받아서 화면에 나타내고 각 상태별로 필터링하여 조회하는 연계 구현 예제입니다.

### 파일 구성
- `index.html`: 메인 html
- `js,js`: 내부 처리 로직
- `server-data.js`: 외부 시스템에서 제공되는 데이터
- `style.css`: 스타일

### 실행 방법
1. 모든 파일을 같은 폴더에 저장하기
2. `index.html` 파일을 웹 브라우저로 열기
3. 탭버튼을 클릭하여 상태별로 학생 카드 조회하기

### 주요 기능
- 외부 학생 관리 서버 데이터 받아서 카드 목록으로 나타냈습니다.
- 카드에는 각 학생의 아이디, 이름, 반, 출석 상태가 보여집니다.
- 필터를 적용하여 각 상태 탭버튼을 클릭 시, 해당 상태의 카드 목록이 보여집니다.
- '출석', '결석', '지각' 이 각각 다른 상태임을 보여주고자 각자 다른 색으로 구분하였습니다. (*출석: 파란색, 결석: 빨간색, 지각: 노란색*)

### 시험 요구사항 설명
#### 외부 서버란?
외부 서버란, 내 컴퓨터가 아닌 다른 곳에 있는 컴퓨터에 저장된 데이터를 말합니다. 브라우저가 서버에 네트워크 요청을 보내면, 서버는 요청된 데이터를 브라우저로 전달하고, 브라우저는 그 내용을 받아 화면에 활용할 수 있습니다. 위 예제에서는 실제로 네트워크 요청을 통해 받은 데이터는 아니지만, `server-data.js`가 그 역할을 대신 수행하고 있다고 볼 수 있습니다. `index.html`에서 `<script src="server-data.js"></script>` 태그를 사용해, 외부 서버에서 제공된 것처럼 구성한 데이터를 내부 웹 시스템에 연결하였습니다.

#### 어떤 데이터가 연계 데이터인지
연계 데이터는 `server-data.js` 파일 내 `serverStudents` 변수에 저장된 배열입니다. 이 배열에는 각 학생의 아이디, 이름, 반, 출석 상태에 대한 정보가 담겨 있습니다. 출석 상태는 ‘출석’, ‘결석’, ‘지각’ 총 3개의 카테고리로 나뉘며, 학생 추가나 기존 정보 수정 등 업데이트가 가능한 데이터입니다. 해당 데이터를 내부 웹 시스템에 연결하여 카드 형식으로 가공한 뒤, 화면에 표시되도록 구현하였습니다.

#### 외부 서버 → 내부 웹 데이터 흐름
데이터 흐름은 다음과 같이 진행됩니다.
1. **외부 학생 서버에서 학생 정보에 대한 데이터를 생성합니다.**<br>
이 예제에서는 `server-data.js` 파일의 `serverStudents`에 데이터가 배열 형식으로 저장되어 있습니다.
2. **외부 서버에서 생성한 데이터를 HTML 파일에 불러옵니다.**<br>
`index.html`에서 `<script src="server-data.js"></script>` 태그를 사용해 데이터를 불러왔습니다.
3. **외부 서버에서 받은 데이터를 내부 로직에서 처리합니다.**<br>
`js.js`에서 외부에서 받은 데이터를 이용해 `studentCard` 함수로 값을 전달하고, 카드 형식으로 가공하여 화면에 보여줍니다. 출석 상태별로 학생 정보 카드를 보여주기 위해 `filter` 함수를 사용하였으며, ‘전체’ 탭을 누르면 모든 정보가 보이고, 각 상태 탭을 누르면 해당 상태의 정보만 표시됩니다.
4. **데이터를 효과적으로 보여줄 수 있도록 가공합니다.**<br>
‘출석’, ‘결석’, ‘지각’ 상태별로 다른 색을 사용하여 상태 정보를 한눈에 확인할 수 있도록 구성했습니다.
5. **사용자가 탭 버튼을 눌러 필터링된 결과를 확인할 수 있습니다.**<br>
화면에 보이는 4개의 탭 버튼 중 하나를 클릭하면, 그에 해당하는 정보들이 필터링되어 나타나며, 왼쪽 상단에서 총 몇 개의 정보가 있는지도 함께 확인할 수 있습니다.

#### 왜 이 구조가 연계인지
이 구조가 연계라고 볼 수 있는 이유는 다음과 같습니다. 외부 데이터 파일인 `server-data.js`에 저장된 데이터를 `index.html`에서 `<script src="server-data.js"></script>` 태그를 통해 불러오고, 이를 `js.js`에서 처리하여 학생 출석 상태를 조회할 수 있도록 구성했기 때문입니다. `js.js`에서는 `server-data.js`의 데이터를 가져와 학생 정보를 확인할 수 있는 카드를 생성하고, 출석 상태에 따라 필터링하여 화면에 보여줍니다. 이로 인해 데이터의 생성, 연결, 가공, 표현이 각각 분리되면서도 하나의 흐름으로 연계되어 동작하는 구조를 이루게 됩니다.

#### 이 예제로 평가 가능한 능력 단위
1. **연계 요구사항 분석 및 데이터 식별**<br>
사용자가 보는 화면에 외부에서 제공되는 데이터 중 어떤 정보를 가져와 보여줘야 하는지, 그리고 이를 어떤 형식으로 표현해야 하는지에 대한 요구사항을 분석하고, 그에 맞는 데이터를 식별할 수 있는지를 평가할 수 있습니다.
2. **데이터 연계 및 활용**<br>
외부 데이터를 활용하여 사용자가 보는 화면에서 정보를 효과적으로 전달할 수 있는 UI를 구현할 수 있는지를 평가할 수 있습니다.
3. **외부 서버와 내부 웹 시스템에 대한 이해**<br>
외부 서버의 데이터를 내부 웹 시스템에서 활용함으로써, 두 시스템이 어떻게 상호작용하는지 이해하고, 각각이 어떤 역할을 수행하는지에 대해 평가할 수 있습니다.
